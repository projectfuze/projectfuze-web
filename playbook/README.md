# Project Fuze — Engagement Playbook

The intellectual and operational backbone behind every Project Fuze engagement. Built on **The Compounding Org** framework — a full-org AI-readiness maturity model that combines Product Intent (David) and Compound Engineering (Lucas).

## How to Use This Playbook

Every engagement follows the same arc: **Scan → Build → Hand Off**. The templates in this directory are designed to be used sequentially within that arc, and to compound across engagements.

## Directory Structure

| Directory | What's Inside | When to Use |
|-----------|--------------|-------------|
| `framework/` | The Compounding Org maturity model, scoring rubric, belief system | Reference throughout every engagement |
| `scan/` | Intake, interview guides, scoring templates, report structure | AI-Readiness Scan (2 weeks) |
| `build/` | Architecture templates, runbooks, training outlines | The Build — Architect or Turnkey (8 weeks) |
| `engineering/` | Quality gates, CI/CD audit, trust ladder, agent-native checklist | Turnkey engagements (Lucas's workstream) |
| `handoff/` | Self-assessment guide, retrospective template | Final 2 weeks of every engagement |
| `compounding/` | Pattern library, benchmarks, agent prompts | Grows after every engagement |

## The Compounding Principle

Every engagement makes the next one faster, sharper, and better:

- Templates sharpen from real-world usage
- Scoring rubric calibrates as we see more orgs
- Agent prompts refine based on output quality
- Industry benchmarks accumulate (after 5+ scans)
- Pattern library grows ("when we see X, root cause is Y")
- Time estimates get more accurate

## Post-Engagement

After every engagement, run the [Retrospective Template](handoff/retrospective-template.md) to capture what compounds.
