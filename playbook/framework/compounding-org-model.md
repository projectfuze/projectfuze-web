**When**: Referenced throughout every engagement — grounds the Scan, informs the Build, frames the Handoff.

## The Universal Loop

Every function, every stage, every cycle follows the same pattern:

**Plan → Execute → Review → Compound**

Each cycle teaches the system something that makes the next one faster. The difference between stages is *how much of this loop agents can handle* vs. *how much requires human intervention*.

---

## The 6 Stages

### Stage 0 — Manual

| Function | What It Looks Like |
|----------|--------------------|
| Product | Tribal knowledge, decisions live in heads |
| Design | Each screen built from scratch |
| Engineering | No AI, relying on docs + Stack Overflow |
| **Org Signal** | **"We just talk about it"** |

No system. No structure. Everything depends on individual memory and ad-hoc communication.

### Stage 1 — Chat-Assisted

| Function | What It Looks Like |
|----------|--------------------|
| Product | ChatGPT for research and drafting |
| Design | AI for exploration and variant generation |
| Engineering | Copilot suggestions, AI for boilerplate |
| **Org Signal** | **"We use AI tools" (but nothing compounds)** |

Individuals use AI, but there's no shared system. Each person's AI usage is siloed. Output doesn't feed back into anything.

### Stage 2 — Agentic, Approve Everything

| Function | What It Looks Like |
|----------|--------------------|
| Product | AI drafts specs, humans rewrite most of it |
| Design | AI generates options, humans redo from scratch |
| Engineering | Agents edit code, humans review every line |
| **Org Signal** | **"We have agents but we review everything"** |

The org has agents but treats them like junior employees who can't be trusted. Every output gets manually reviewed, rewritten, or redone. The bottleneck is human review capacity.

**This is where most orgs are stuck.**

### Stage 3 — Plan-First, Review Output *(The Target)*

| Function | What It Looks Like |
|----------|--------------------|
| Product | Structured intent drives agent drafts |
| Design | Design system constrains, agents produce within it |
| Engineering | Plans drive execution, review PRs not individual lines |
| **Org Signal** | **"We direct. Agents transmit."** |

The critical shift: humans define *what* and *why*. Agents handle *how*. Review moves from line-by-line to artifact-level because the system has quality gates built in.

**The 2→3 transition is the money move. It's organizational, not technical. This is what Project Fuze sells.**

### Stage 4 — Idea to Artifact

| Function | What It Looks Like |
|----------|--------------------|
| Product | Outcome description → decision artifact |
| Design | Brief → tested prototype |
| Engineering | Idea → pull request |
| **Org Signal** | **"We describe, agents deliver"** |

The gap between intent and artifact shrinks to near-zero. Humans describe outcomes; the system produces verified artifacts.

### Stage 5 — Parallel Fleet

| Function | What It Looks Like |
|----------|--------------------|
| Product | Parallel initiative streams running simultaneously |
| Design | Parallel explorations with rapid convergence |
| Engineering | Parallel cloud execution across workstreams |
| **Org Signal** | **"Multiple workstreams, no bottleneck"** |

The org runs multiple parallel workstreams without proportional headcount. The system scales horizontally because the backbone handles coordination.

## Why the 2→3 Transition Matters

Most orgs plateau at Stage 2 because:

1. **Intent isn't structured enough** — Agents can't act on vague specs, Slack threads, or tribal knowledge
2. **Quality gates don't exist** — Without automated verification, humans must review everything manually
3. **Handoffs leak signal** — Every unstructured handoff between functions loses context
4. **The backbone is missing** — No system connects Product decisions to Engineering execution

The fix is organizational infrastructure, not better tools. You need:
- Structured decision architecture (Product Intent)
- Quality gates and trust ladders (Compound Engineering)
- Connected workflows where context flows automatically

This is exactly what Project Fuze installs.
