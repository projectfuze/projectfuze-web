# Intake Packet

**What**: Pre-engagement questionnaire sent to the client after contract signing.
**When**: Immediately after Scan contract is signed, before Week 1.
**Who**: David sends. Client fills out (typically founder or Head of Product).
**Output**: Completed packet providing access and baseline context for the Scan.

---

## Email Template

Subject: **Project Fuze — AI-Readiness Scan: Getting Started**

Hi [Name],

Welcome aboard. Before we start the Scan, we need a few things to make Week 1 productive. Please complete the following by [date — 3 business days before Week 1]:

---

### 1. Org Structure

- [ ] Org chart or team structure document
- [ ] Number of employees (total and by function: Product, Design, Engineering, other)
- [ ] Key people we'll interview (names + roles for: Founder/CEO, Head of Product, Head of Eng, Design Lead)

### 2. Tool Stack

- [ ] List of all tools used for: project management, communication, design, development, CI/CD, documentation
- [ ] Which tools have AI features enabled?
- [ ] Any AI-specific tools in use (Copilot, Cursor, ChatGPT Team, etc.)

### 3. Access (Guest/Viewer Only)

- [ ] Slack — guest access to relevant channels
- [ ] PM tool (Linear, Jira, Asana, etc.) — viewer access
- [ ] Code repository — read-only access (for Lucas's engineering audit)
- [ ] CI/CD dashboard — viewer access
- [ ] Calendar access for key people (to schedule interviews)

### 4. Artifacts

- [ ] 3–5 recent product specs or PRDs (representative, not cherry-picked)
- [ ] 3–5 recent pull requests or engineering tickets (representative)
- [ ] Any existing process documentation (how we ship, how we decide, etc.)

### 5. Permission

- [ ] Permission to record interview sessions (audio only, for AI-assisted transcript processing)

### 6. The One Question

Please answer in 2–3 sentences:

> **"What's the #1 thing you'd change about how your team works?"**

---

This typically takes 30–60 minutes to assemble. If any item is blocked, let me know and we'll work around it.

Looking forward to getting started.

— David

---

## Processing

When the intake packet arrives:

1. Review org chart → prepare interview questions tailored to org size and structure
2. Review tool stack → identify integration points and gaps for Lucas
3. Verify all access is working before Week 1
4. Read the artifacts → form preliminary hypotheses
5. Note the "one question" answer → this often reveals the core pain point
6. Share relevant items with Lucas (repo access, CI/CD, tool stack, engineering artifacts)
